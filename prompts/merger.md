# Merger Rules

Use these rules when merging new material into an existing self skill.

## Priority Order

1. Direct user correction
2. Repeated new evidence
3. Repeated old evidence
4. Old one-off examples
5. Weak inference

## Merge Strategy

- Update only the affected section. Do not rewrite unrelated sections.
- If the new material changes audience-specific behavior, scope the rule by audience instead of replacing the global rule.
- If a new pattern is real but narrow, add it as a conditional rule.
- Preserve useful contradictions when they are context-specific.

## Do Not Do

- Do not flatten all tones into one "average" voice.
- Do not overwrite hard boundaries with a single casual example.
- Do not delete old evidence unless the user explicitly says the old rule is wrong now.

## Output Format

For each merge, produce:

```text
变更位置：
旧规则：
新规则：
证据：
是否影响 Work / Persona / Both：
```
